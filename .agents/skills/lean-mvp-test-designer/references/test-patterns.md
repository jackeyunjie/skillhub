# Test Patterns

## Problem Interview

Use when:
- the problem is not clearly understood
- buyer workflow is unknown
- language and urgency need discovery

Observe:
- concrete past behavior
- current workaround
- budget owner
- urgency and timing

Avoid:
- asking "would you use this?"

## Fake Door / Smoke Test

Use when:
- action intent is uncertain
- a low-fidelity front door can be shown

Observe:
- click
- signup
- request access
- leave contact
- reply with concrete interest

Avoid:
- measuring only impressions or likes

## Concierge Test

Use when:
- value and delivery can be manually simulated
- automation is not yet necessary

Observe:
- whether the user still wants the result after manual delivery
- time and quality bottlenecks
- review burden

## Paid Pilot / Preorder

Use when:
- willingness to pay is central
- buyer has enough trust to make a small commitment

Observe:
- deposit
- paid pilot
- contract discussion
- budget process

## Internal Workflow Pilot

Use when:
- validating an AI workflow, SOP, or operational change

Observe:
- time saved after review
- error rate
- handoff clarity
- whether users keep using it unprompted
