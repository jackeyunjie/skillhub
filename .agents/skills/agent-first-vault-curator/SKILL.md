# Agent First Vault Curator

Use this as the workspace routing skill for this vault.

## Core Capabilities

1. Turn repeated tasks into reusable skill packages.
2. Split material into `Concepts/`, `Workflows/`, `Skills/`, `Agents/`, and `Samples/`.
3. Decide whether a task should stay as a note or become a callable skill.
4. Route execution to the more specific local skills in this workspace.

## Routing Rules

- For repeated scheduled tasks, read [../codex-automation-briefing/SKILL.md](../codex-automation-briefing/SKILL.md).
- For spreadsheet-to-lookup products, read [../codex-query-webapp/SKILL.md](../codex-query-webapp/SKILL.md).
- For source-to-script content work, read [../codex-content-factory/SKILL.md](../codex-content-factory/SKILL.md).
- For post-recording cleanup, read [../codex-talking-head-video-edit/SKILL.md](../codex-talking-head-video-edit/SKILL.md).
- For vault capture and note splitting, read [../codex-knowledge-base-capture/SKILL.md](../codex-knowledge-base-capture/SKILL.md).
- For broad Agent First task redesign, read [../codex-agent-first-workflow/SKILL.md](../codex-agent-first-workflow/SKILL.md).

## Operating Rules

- Bias toward skill packaging when the task will be reused.
- Keep notes as explanation and examples; keep skills as callable operating contracts.
- Do not duplicate large reference material across packages.
- Save one concrete sample whenever a workflow has been proven in practice.
