# Risk Structures

Use risk structure to avoid applying the same validation method to every case.

## ToC / Fast Consumer Behavior

Common first risks:
- attention
- desire
- click / signup / purchase conversion
- repeated use

Good early evidence:
- behavior in a realistic context
- comparison against alternatives
- willingness to take a next step

## ToB / High-Value Long Cycle

Common first risks:
- whether the pain is important enough
- whether the buyer has budget and authority
- whether switching cost blocks adoption
- whether the value case is provable

Good early evidence:
- deep interviews with target buyers
- concrete workflow details
- commitment to a next meeting, pilot, or data sharing

Do not overvalue shallow clicks.

## Content / Course / Knowledge Product

Common first risks:
- product kernel
- user urgency
- perceived authority / trust
- completion and transformation

Good early evidence:
- users voluntarily compare it favorably against alternatives
- users finish a small sample and ask for more
- users can describe the value in their own words

## Hardware / Operationally Heavy Product

Common first risks:
- technical feasibility
- irreversible investment
- supply chain or delivery risk
- substitute with lighter solution

Good early evidence:
- low-fidelity prototype
- simulation
- concierge workflow
- paid reservation only after core risk is reduced

## AI Workflow / Automation

Common first risks:
- task repeatability
- input stability
- review burden
- error cost
- integration friction

Good early evidence:
- repeated cases with same input-output contract
- measurable time saved after review
- clear failure boundaries
