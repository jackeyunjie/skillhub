# Business Formula Core Contradiction Finder

Use this before picking actions. The job is to separate the surface symptom from the real constraint.

## Workflow

1. State target, current state, and gap.
2. List visible symptoms.
3. Identify the popular explanation people are likely chasing.
4. Test whether the explanation actually matches the facts.
5. Name the core contradiction.
6. Explain why it is more important than the surface symptom.
7. Recommend the formula family or parameter layer to decompose next.

## Output Shape

```text
Target:
Current state:
Gap:
Surface symptoms:
Likely false contradiction:
Core contradiction:
Why this is the real bottleneck:
Evidence needed:
Next formula family / parameter layer:
First action to clarify:
```

## Rules

- Do not start from department ownership.
- Do not accept "traffic is not enough" unless traffic quality and conversion chain have been checked.
- Prefer one core contradiction over a broad problem list.
- Treat "conversion is low" as a symptom until the failing node is named.
- If the data is insufficient, state the minimum evidence needed.

## Common False Contradictions

- "Need more traffic" when trust or offer is weak.
- "Need a discount" when problem urgency is weak.
- "Need more sales calls" when product usage is shallow.
- "Need more features" when customer success is missing.
- "Need better marketing" when the target segment is wrong.
