# Business Formula Skill

> 一堂方法论：用业务公式把业务拆解、假设管理、验证迭代串成一套可执行的操盘方法。

## 核心工作流（Ω 流程）

```
明确目标 → 做加法（拆公式+提假设） → 做减法（筛关键假设） → 做验证 → 不断调整
```

### Step 1：判断业务类型

先判断业务属于哪一类，决定公式拆法：

| 类型 | 核心瓶颈 | 关注公式 |
|------|---------|---------|
| **单次成交型** | 怎么让客户从触达到成交 | 收入提升类 + 连续动作型/动作节点型 |
| **持续复购型** | 留得住、留得久 | 竞争提升类（脱离成本）+ 留存公式 |

> 不要上来就拆 GMV = 流量×转化×客单价×复购，拆完还是不知道该先做什么。

### Step 2：确定目标与核心矛盾

- 要提升什么？提升到多少？有什么约束？
- 与 benchmark/竞对对比，差距在哪？
- 当前数据 vs 目标数据，最大缺口在哪一层？

### Step 3：拆解公式（先切分 +，再转化 ×）

**正确顺序：**
```
第一步：先切分（+）
  GMV = A渠道GMV + B渠道GMV + C渠道GMV
  或 GMV = 新用户GMV + 复购用户GMV + 转介绍GMV

第二步：再拆转化（×）
  A渠道GMV = 曝光量 × 点击率 × 转化率 × 客单价
  新用户GMV = 曝光 × 点击 × 落地页完成率 × 支付转化率 × 首单客单价
```

**为什么先切分？**
- 不同渠道/用户类型的转化逻辑可能完全不同
- 不切分，你看不到差异、分不清优先级、动作会打架

**十大公式范式速查**（详见 [references/formula-patterns.md](references/formula-patterns.md)）：
- 收入提升类：流量广告、APP推广、电商获客、企业咨询、SaaS转化、加盟商、线下零售、用户生命周期
- 竞争提升类：黏性/脱离成本、规模化生产
- 运营提升类：留存节点、续费/复购、转化漏斗、高客单价长链路

### Step 4：深挖参数到 L3-L4

**参数冰山模型**（6 层，详见 [references/parameter-iceberg.md](references/parameter-iceberg.md)）：

| 层级 | 名称 | 能否指导动作 |
|------|------|-------------|
| L1 | 基础财务参数（GMV/收入/成本） | ❌ 只是科目 |
| L2 | 业务科目参数（流量/转化率/客单价） | ❌ 正确但无用 |
| L3 | 可行动指标参数（切分后的细分） | ✅ 能指导动作 |
| L4 | 定性参数的可定量化 | ✅ 可验证 |
| L5 | 本质洞察参数 | ✅ 理解"为什么" |
| L6 | 魔法参数 | ✅ 战略级认知 |

**目标：至少拆到 L3-L4**，参数变成可行动的指标。

### Step 5：定性参数定量化

**每个定性参数，必须找到 3-5 个可定量的行为指标来佐证。**

常见定性参数 → 行为指标示例：

| 定性参数 | 可定量的行为指标 |
|---------|----------------|
| 信任度 | 页面停留时长(>3min)、详情页下滑完成率(>80%)、试听课完课率(>70%)、咨询发起率(>15%) |
| 使用深度 | 月活账号率、核心功能使用率(>50%)、关键角色参与度、周活跃天数(>3天) |
| 价值感知 | 学习数据导出次数(>2次/月)、报告打开率(>60%)、推荐使用部门数(>3个) |
| 切换成本 | 历史数据积累量(>1年)、使用习惯养成率(>70%)、定制化功能数量(>5个) |
| 满意度 | 评价提交率(>40%)、评分均值(>4.5)、差评回复后二次到店率(>30%) |
| 场景绑定 | 工作日午餐占比(>60%)、家庭聚餐占比(>20%)、外卖频次(>2次/周) |

> 拆完之后，"提升信任度"不再是正确但无用的废话，而是：优化详情页结构，目标下滑完成率从 45%→80%。

### Step 6：判断逻辑关系

**详见 [references/logic-relationships.md](references/logic-relationships.md)**

核心就三件事：

1. **先切分再转化**（顺序不能错）
2. **运算符号要对**（+ vs ×）
   - "+" = 叠加关系，有一个就够了，多了更好，不要平均用力
   - "×" = 漏斗关系，缺一不可，必须逐个提升
3. **关系本质要对**（相关 vs 因果）
   - 相关：A 和 B 一起变化，但 A 不一定导致 B → 只能看趋势
   - 因果：A 变化 → B 一定变化，控制变量后依然成立 → 才能做决策

### Step 7：建立假设与数据埋点

每个可行动的参数，对应：
- **假设**：做什么动作？预期提升到什么数值？
- **数据埋点**：用什么指标验证？当前基线是多少？
- **验证方式**：数据复盘 / 内部评估 / 用户调研 / A/B 实验

### Step 8：形成假设池与验证计划

**假设池基础字段：**

| 字段 | 说明 |
|------|------|
| 提出 | 提出人 + 假设内容 |
| 优先级 | 价值高低 + 成本高低 |
| 负责 | 验证人 |
| 结果 | 验证结果和认知 |

**进阶管理框架：**
```
         ┌─────────────┐
         │   业务公式   │
         └──────┬──────┘
    ┌───────────┼───────────┐
    ↓           ↓           ↓
┌───────┐  ┌─────────┐  ┌─────────┐
│ 假设池  │  │ 关键会议  │  │ 仪表盘   │
└───────┘  └─────────┘  └─────────┘
```

## 常见陷阱检查清单

在帮用户拆解业务公式时，逐项检查：

- [ ] **只拆了一组公式？** → 一定有盲区，按业务类型多拆几组
- [ ] **停在 L1-L2？** → 参数只是"科目"不是"抓手"，继续往下拆
- [ ] **有定性参数没定量？** → 每个定性参数找 3-5 个行为指标
- [ ] **一上来就用乘法？** → 应该先切分（+），再拆转化（×）
- [ ] **运算符号凭感觉写？** → 判断是叠加（+）还是漏斗（×）
- [ ] **把相关当因果？** → 需要数据支撑、控制变量、实验验证
- [ ] **假设没有数据埋点？** → 无法验证，无法迭代
- [ ] **没有假设池管理？** → 想法一堆，丢三落四
