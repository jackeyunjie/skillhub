# 逻辑关系判断指南

> 来源：一堂业务公式课程 - 孔源专家分享 + 逻辑关系实操课
> 参数关系要对，才能做对决策

---

## 核心问题

你拆业务公式的时候，参数之间的关系是怎么确定的？
- 凭感觉写"×"或"+"？
- 有意识地区分相关和因果？
- 能定量判断每个参数的数值区间？

**确定参数关系，比拆参数本身难得多。**
- 拆参数只需要"知道业务"
- 确定关系需要"验证因果"——证明 A 变化会导致 B 变化，控制其他变量后依然成立

## 技巧一：先切分，再拆转化（拆解顺序）

### ❌ 错误顺序
```
GMV = 流量 × 转化率 × 客单价 × 复购率
```

### ✅ 正确顺序
```
第一步：先切分（+）
GMV = A渠道GMV + B渠道GMV + C渠道GMV
  或
GMV = 新用户GMV + 复购用户GMV + 转介绍GMV

第二步：再拆转化（×）
A渠道GMV = 曝光量 × 点击率 × 转化率 × 客单价
新用户GMV = 曝光 × 点击 × 落地页完成率 × 支付转化率 × 首单客单价
```

### 为什么先切分？
- 不同渠道、不同用户类型的转化逻辑**可能完全不一样**
- 活跃用户的复购逻辑，和沉默用户的唤醒逻辑，公式不同
- A渠道的转化逻辑，和B渠道的转化逻辑，公式不同
- 不切分，你看不到差异、分不清优先级、动作会打架

### 三类用户的增长逻辑差异

| 用户类型 | 核心矛盾 | 优化动作 |
|---------|---------|---------|
| **新用户** | 流量够不够？转化漏斗哪一环掉了？ | 优化投放渠道、优化落地页、优化首单体验 |
| **复购用户** | 用户用得爽不爽？有没有触发复购的动力？ | 提升产品体验、设计复购机制、建立会员体系 |
| **转介绍用户** | 用户愿不愿意分享？分享有没有激励？ | 优化分享体验、设计激励机制、降低分享门槛 |

## 技巧二：判断运算符号（+ vs ×）

### 叠加关系（+）

**特征：** 多个要素共同作用，缺一个不影响整体，多一个更好

**典型示例：**
```
信任度 = 信任背书1 + 信任背书2 + 信任背书3
满意度 = 环境满意度 + 教练满意度 + 前台服务满意度
收入 = 渠道A收入 + 渠道B收入 + 渠道C收入
核心功能使用率 = 功能A使用率 + 功能B使用率 + 功能C使用率
```

**判断方法：** 共同影响要素，有一个就够了

### 漏斗关系（×）

**特征：** 每一步都在"过筛子"，任何一个环节掉链子，整个链条就断了

**典型示例：**
```
转化率 = 触达率 × 打开率 × 点击率 × 下单率 × 支付率
使用深度 = 注册率 × 激活率 × 核心功能使用率 × 高频使用率
续卡率 = 到店频率 × 满意度 × 价格接受度
活跃客户续费率 = 使用深度 × 价值感知 × 切换成本
```

**判断方法：** 单一影响要素，每一步都在过筛子

### 写错符号的后果

| 错误 | 后果 |
|------|------|
| "+"写成"×" | 觉得"每一个都要做到完美"，导致平均用力，找不到重点 |
| "×"写成"+" | 觉得"缺一个也没关系"，导致关键环节被忽略，业务出问题 |

### ToB SaaS 续费率示例

```
活跃客户续费率 = 使用深度 × 价值感知 × 切换成本（×，漏斗，缺一不可）

使用深度 = 功能A使用率 + 功能B使用率 + 功能C使用率（+，叠加，有一个就够了）
价值感知 = 效果数据化 + 成功案例 + 报告（+，叠加）
切换成本 = 数据沉淀 + 工作流嵌入 + 团队习惯（+，叠加）
```

**优先级判断：**
- "×"关系：使用深度、价值感知、切换成本，三者缺一不可
  - 核心矛盾：这三个维度，哪个最弱？
- "+"关系：使用深度内部，ABC三个功能，有一个就够了
  - 核心矛盾：哪个最容易让客户用起来？

> 运算符号判断对了，优先级才能判断对。

## 技巧三：从相关到因果，判断关系本质

### 相关性

**定义：** A 和 B 一起变化，但 A 不一定导致 B

**典型示例：**
| 表面相关 | 真相 |
|---------|------|
| 投放量增加，转化率下降 | 真正原因是"流量质量下降" |
| 功能使用率提高，续费率提高 | 真正原因是"客户成功体系" |
| 到店频率和续卡率正相关 | 真正因果链是满意度→到店频率→到店习惯→续卡率 |
| 冰激凌销量 vs 溺水事故 | 两者都受气温影响（共同原因） |

**价值：** 只能帮你"看趋势"

### 因果性

**定义：** A 变化 → B 一定变化，控制其他变量后 A 仍是 B 的原因

**典型示例：**
| 因果关系 | 说明 |
|---------|------|
| 核心功能使用率提高 → 续费率提高 | "使用深度"因果链中的一环 |
| 落地页完成率提高 → 支付转化率提高 | 看完落地页的人才会支付 |
| 满意度 → 到店频率 → 到店习惯 → 续卡率 | 满意度是"因的因"，直接抓手是"到店习惯" |

**价值：** 才能帮你"做决策"

### 常见陷阱

**陷阱：把"因的因"当成"直接的因"**

健身房续卡率：
```
满意度 → 到店频率 → 到店习惯 → 续卡率
```

- 满意度是"因的因"，不是"直接的因"
- 如果你把"满意度"当成"续卡率"的直接原因，会拼命提升满意度
- 但满意度阈值不会再往上走，损耗越来越高
- **真正该做的：把"到店频率"变成"到店习惯"**

### 判断因果 vs 相关的三种方法

| 方法 | 说明 |
|------|------|
| **常识判断** | 是否符合基本逻辑和常识 |
| **逻辑推理** | 是否存在第三变量、反向因果等混淆因素 |
| **控制变量测试** | 通过实验验证，控制其他变量 |

## 参数关系三条小规律

1. **拆解顺序要对：先切分再拆转化**
   - GMV = A+B+C，然后 A = a1×a2×a3
   - 不切分，直接拆"总公式"，拆出来的参数关系对谁都不准

2. **运算符号要对："+"和"×"不能乱写**
   - "+" = 叠加关系，有一个就够了，不要平均用力
   - "×" = 漏斗关系，缺一不可，必须逐个提升
   - 写错符号 = 优先级判断错误

3. **关系本质要对：相关不是因果**
   - L2 的相关关系，只能帮你"看趋势"
   - L3 的因果关系，才能帮你"找到抓手"
   - 跨越 L2→L3 的方法：控制其他变量，看 A 是不是 B 的真正原因
